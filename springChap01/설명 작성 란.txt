
HTTP (Hypertext Transfer Protocol)
	클라이언트와 서버 간의 통신을 위한 프로토콜
	주로 웹에서 데이터를 주고받을 때 사용

HTTP 구조
	1. 메서드(method)
		GET
		POST : 데이터베이스나 서버에 데이터를 제출하고싶을 때 사용
	
	2. 헤더(Header)
		인증 토큰이나 캐시, 사용자 정보와같은 내용은 meta나 데이터 헤더 부분을 이용해서 전달
		눈에 보이지는 않지만 중요한 정보를 담을 때 주로 사용
		
	3. 바디(Body)
		바디에는 주로 <input/>이나 <text/>, <textarea/> 등등
		바디에 작성하는 태그 내용을 DB에 실제로 전송되는 데이터를 포함
	
RESTful
	REST 아키텍처 스타일을 따르는 웹 서비스

	자원(Resource) : 
		url 주소에 담겨있는 모든 정보들
	표현(Representation) : 
		url로 표현하는 자원이 다양한 방식으로 표현될 수 있음
		JSON, XML 이 대표적인 표현법 중 하나
	상태전이(Stateless) :
		서버는 서버일 뿐, 각 요청을 저장하지 않음 
	통합 인터페이스(Uniform Interface) :
		인터페이스는 개발자가 지정해준 디자인을 가지고있으며, 
		컴퓨터가 임의로 데이터를 변경하지 않고 화면 데이터를 일관성 있게 제공
